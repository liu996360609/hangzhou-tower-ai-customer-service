## 1. V2方案概述与核心变更

### 1.1 架构决策

基于对阿里云百炼平台能力的全面评估，本项目采用 **"平台托管 + 自研补充"** 的混合架构：

| 层次 | 实现方式 | 说明 |
|------|----------|------|
| **智能体应用 / 工作流应用** | 百炼平台托管 | 通过百炼控制台配置智能体（Prompt、模型选择、工具挂载、知识检索组件），通过API调用运行 |
| **知识库底层能力** | 百炼API + 自研管理后台 | 利用百炼知识库API完成文档存储、解析、分片、向量化、检索，自研后台复刻完整的管理UI |
| **会话管理、路由、报表** | 自研后端 | 会话路由、坐席管理、会员路由、排队机制、数据报表、**VIP专属路由**、**多模态推荐引擎**等完全自研 |
| **前端接入层** | 自研前端 | 小程序、H5、企业微信等渠道前端自研，**新增专属客服小程序工作台** |

### 1.2 V2核心变更说明

V2方案相较于V1的核心变化集中在 **用户分流策略** 和 **服务响应模式**：

#### 1.2.1 常规用户流程变更

| 变更点 | V1 | V2 |
|--------|----|----|
| AI响应 | 纯文本 | **多模态响应（文本 + 导航 + 商户/商品/优惠券/活动推荐卡片）** |
| 转人工 | 在线客服转接 | **仅提供服务台电话 + 服务台地址导航，不再支持在线人工聊天** |
| 推荐能力 | 无 | **新增后端多模态推荐引擎，从商户/商品/优惠券/活动/导航数据库聚合推荐数据** |

**常规用户流程**：
```
用户提问 → AI识别意图 → RAG检索知识库 → 多模态响应组装（文本 + 导航卡片 + 商户/商品/优惠券/活动推荐卡片）→ 返回用户
    │
    └── 若用户要求人工 → 展示服务台电话号码 + 服务台地址导航引导（无在线转接）
```

#### 1.2.2 VIP用户流程（全新设计）

**VIP用户流程**：
```
VIP首次提问 → VIP身份识别 → 查询专属客服状态
    │
    ├── 专属客服空闲 → 直接分配 → 专属客服通过小程序工作台接管
    │
    └── 专属客服忙碌 → 展示排队位置 + 两个选项：
            ├── 选项A：排队等待
            └── 选项B：拨打专属客服电话
```

**连接后**：VIP用户的所有后续提问均由专属人工客服通过小程序工作台处理，不再经过AI。

#### 1.2.3 专属客服小程序工作台（全新）

| 能力 | 说明 |
|------|------|
| **消息推送** | VIP用户提问时，实时推送通知给专属客服 |
| **VIP信息面板** | 查看VIP用户档案（会员等级、积分、历史消费、偏好标签） |
| **对话管理** | 查看VIP用户问题、历史对话记录 |
| **多模态回复** | 客服可发送文本 + 推送商户/商品/活动/优惠券/导航数据卡片 |
| **后端数据库支撑** | 商户DB、商品DB、活动DB、优惠券DB、导航DB，客服从这些数据库选择数据推送给用户 |

### 1.3 方案核心原则

1. **智能体/工作流不自建**：所有Agent和Workflow的配置、编排、模型绑定均在百炼控制台完成，后端仅调用运行API
2. **知识库管理自研复刻**：通过百炼数据连接API和知识库API，完整复刻百炼控制台的文档上传、解析、分片管理、检索测试等功能
3. **会话层完全自研**：百炼平台仅提供AI能力，会话管理、转人工、排队、路由等业务逻辑全部自研
4. **前端多渠道自研**：小程序、H5、企微等接入渠道前端自研，后端统一路由
5. **V2新增：常规/ VIP双通道**：常规用户走AI多模态推荐，VIP用户走专属客服直连
6. **V2新增：客服小程序工作台**：为专属客服提供VIP用户服务的工作台端
7. **V2新增：多模态推荐引擎**：从后端业务数据库聚合商户/商品/优惠券/活动/导航数据

### 1.4 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 百炼平台 | 智能体应用 + 知识库 + DashScope API | AI核心引擎 |
| 大模型 | Qwen 3（通义千问3） | 主力对话模型 |
| Embedding | text-embedding-3 | 向量模型（百炼内置） |
| 自研后端 | Java Spring Boot 3 + Spring Cloud Alibaba | 业务逻辑、会话路由、管理后台、**多模态推荐引擎** |
| 自研前端（管理端） | Vue 3 + Element Plus | 管理后台 |
| 自研前端（移动端） | Vant 4 + 微信小程序 | C端用户对话、**专属客服小程序工作台** |
| 数据库 | MySQL 8.0（RDS） | 业务数据存储 |
| 缓存 | Redis 集群 | 会话状态、热点数据、排队计数、**VIP状态** |
| 消息队列 | RocketMQ | 异步事件驱动、削峰填谷、**VIP推送通知** |
| 对象存储 | 阿里云 OSS | 文件暂存、富媒体资源 |
| 推送服务 | 微信小程序订阅消息 + 极光推送 | **专属客服消息推送** |

## 2. 百炼平台API全面遍历评估

### 2.1 数据连接（原应用数据）API

数据连接API用于管理百炼平台中存储的文档文件，是知识库的数据来源层。

| API | 功能 | 参数概要 | 限流 | 说明 |
|-----|------|----------|------|------|
| **AddCategory** | 新增类目 | `workspace_id`, `category_id`, `name` | 10次/秒 | 每个业务空间最多500个类目 |
| **ListCategory** | 类目列表 | `workspace_id`, `category_ids`（可选） | 10次/秒 | 获取类目详细信息 |
| **DeleteCategory** | 删除类目 | `workspace_id`, `category_id` | 10次/秒 | 永久删除，不可恢复 |
| **ApplyFileUploadLease** | 申请文件上传租约 | `category_id`, `workspace_id`, `file_name`, `md_5`, `size_in_bytes` | 10次/秒 | 返回预签名URL和上传头信息 |
| **AddFile** | 添加文件 | `workspace_id`, `lease_id`, `parser`, `category_id` | 10次/秒 | 将临时存储文件导入数据连接 |
| **AddFilesFromAuthorizedOss** | 从授权OSS导入文件 | `workspace_id`, `category_id`, `parser`, `oss_file_path` | 10次/秒 | 适合批量导入场景 |
| **ListFile** | 文件列表 | `category_id`, `workspace_id`, 分页参数 | 10次/秒 | 获取文档列表及概要信息 |
| **DescribeFile** | 查询文件状态 | `workspace_id`, `file_id` | 5次/秒 | 查询解析状态（INIT/PARSING/PARSE_SUCCESS/PARSE_FAILED） |
| **UpdateFileTag** | 更新文件标签 | `workspace_id`, `file_id`, `tags` | 10次/秒 | 单个文件标签更新 |
| **BatchUpdateFileTag** | 批量更新文档标签 | `workspace_id`, `file_ids`, `tags` | 10次/秒 | 批量操作，适合标签管理 |
| **DeleteFile** | 删除文件 | `workspace_id`, `file_id` | 10次/秒 | 永久删除，不支持删除数据表 |
| **DeleteFiles** | 批量删除文件 | `workspace_id`, `file_ids` | 10次/秒 | 批量删除操作 |
| **GetParseSettings** | 获取类目解析设置 | `workspace_id`, `category_id` | 10次/秒 | 查询当前类目解析配置 |
| **GetAvailableParserTypes** | 获取支持的解析器类型 | `workspace_id`, `file_type` | 10次/秒 | 根据文件扩展名返回可用解析器列表 |
| **ChangeParseSetting** | 修改类目解析设置 | `workspace_id`, `category_id`, `file_type`, `parser` | 10次/秒 | 配置特定文件类型的解析方式 |
| **AddTable** | 添加表格 | `workspace_id`, `connector_id`, 表格信息 | 10次/秒 | 为表格数据连接器添加表格 |
| **UpdateTableFromAuthorizedOss** | 从授权OSS更新表格 | `workspace_id`, `connector_id`, `oss_file_path` | 10次/秒 | 更新连接器中的表格数据 |
| **AddConnector** | 新增连接器 | `workspace_id`, `connector_name`, `connector_type` | 10次/秒 | 当前仅支持文件类型连接器 |
| **GetConnector** | 获取连接器信息 | `workspace_id`, `connector_id` | 10次/秒 | 获取连接器详细信息 |
| **UpdateConnector** | 编辑连接器 | `workspace_id`, `connector_id`, 更新参数 | 10次/秒 | 编辑连接器配置 |
| **DeleteConnector** | 删除连接器 | `workspace_id`, `connector_id` | 10次/秒 | 删除连接器 |

**解析器类型（Parser）**：
- `DASHSCOPE_DOCMIND`：大模型文档解析（适用于PDF/DOC/DOCX，提取能力最强）
- `MARKDOWN`：Markdown解析
- `GENERAL`：通用文本解析
- `QWEN_VL`：多模态视觉解析（适用于图片/PNG/JPG）
- `TABLE`：表格解析

### 2.2 知识库 API

知识库API是RAG检索的核心，提供知识库的创建、索引、检索、分片管理等能力。

| API | 功能 | 参数概要 | 限流 | 说明 |
|-----|------|----------|------|------|
| **CreateIndex** | 创建知识库 | `workspace_id`, `name`, `structure_type`, `source_type`, `sink_type`, `document_ids` | 10次/秒 | 支持非结构化（文档/音视频）和结构化（数据查询/图片问答）两类知识库 |
| **GetIndexJobStatus** | 查询知识库创建任务状态 | `workspace_id`, `index_id`, `job_id` | 10次/秒 | 状态：RUNNING/COMPLETED/FAILED |
| **SubmitIndexJob** | 提交知识库创建任务 | `workspace_id`, `index_id` | 10次/秒 | 完成知识库创建流程 |
| **SubmitIndexAddDocumentsJob** | 提交知识库追加任务 | `workspace_id`, `index_id`, `document_ids` | 10次/秒 | 向已有知识库追加文件 |
| **Retrieve** | 检索知识库 | `workspace_id`, `index_id`, `query`, `top_k`（可选）, `filters`（可选） | **5次/秒** | 核心检索接口，返回召回文档和分数 |
| **ListIndexDocuments** | 查询知识库文件列表 | `workspace_id`, `index_id`, 分页参数 | 10次/秒 | 获取知识库中的文件概要 |
| **ListIndexFileDetails** | 查询知识库文件详情 | `workspace_id`, `index_id`, 分页参数 | 10次/秒 | 获取文件详细信息 |
| **UpdateIndex** | 更新知识库 | `workspace_id`, `index_id`, 更新参数 | 10次/秒 | 更新知识库配置 |
| **DeleteIndexDocument** | 删除知识库下的文件 | `workspace_id`, `index_id`, `document_id` | 10次/秒 | 从知识库中移除文件 |
| **ListIndices** | 查询知识库列表 | `workspace_id`, 分页参数 | 10次/秒 | 获取业务空间下知识库列表 |
| **DeleteIndex** | 删除知识库 | `workspace_id`, `index_id` | 10次/秒 | 永久删除知识库 |
| **ListChunks** | 查询索引下的分片列表 | `workspace_id`, `index_id`, `document_id`（可选）, 分页参数 | 10次/秒 | 查看文本切片列表及信息 |
| **UpdateChunk** | 修改切片 | `workspace_id`, `index_id`, `chunk_id`, `content`（可选）, `title`（可选）, `enabled`（可选） | 10次/秒 | 可修改内容和标题，设置是否参与检索 |
| **DeleteChunk** | 删除切片 | `workspace_id`, `index_id`, `chunk_id` | 10次/秒 | 删除后无法恢复，不参与检索 |
| **GetIndexMonitor** | 获取知识库监控数据 | `workspace_id`, `index_id`, `start_time`, `end_time` | 10次/秒 | 存储监控 + QPS监控（峰值/总数/成功/失败/限流） |

### 2.4 智能体/工作流调用 API

智能体应用和工作流应用**没有创建/编辑API**，仅支持调用已创建的应用。

#### 智能体应用调用

| 方式 | 接口 | 说明 |
|------|------|------|
| **OpenAI兼容接口** | `POST /v1/chat/completions` | 兼容OpenAI SDK，推荐用于简单对话场景 |
| **DashScope原生接口** | `POST /api/v1/services/aigc/text-generation/generation` | 百炼原生接口，功能最完整 |
| **Assistant API** | `POST /v1/assistants/{assistant_id}/threads` 等 | 支持工具调用、长期记忆、流式输出 |

**关键特性**：
- 支持流式输出（SSE），适合实时对话场景
- 支持工具调用（Function Calling），可挂载知识库检索、自定义函数等
- 支持多轮对话（通过messages数组维护上下文）
- 智能体创建、编辑、配置**仅支持控制台操作**，无API

#### 工作流应用调用

| 方式 | 接口 | 说明 |
|------|------|------|
| **工作流运行API** | `POST /api/v1/apps/{app_id}/completion` | 运行已发布的工作流应用 |
| **流式调用** | 同上加 `stream: true` | 支持流式输出工作流中间结果 |

**关键特性**：
- 工作流创建、编排、节点配置**仅支持控制台操作**，无API
- 调用时需传入工作流定义的输入参数
- 支持同步和异步两种调用模式

### 2.6 大模型推理 API（DashScope）

| 接口 | 说明 |
|------|------|
| `POST /v1/chat/completions`（OpenAI兼容） | 标准Chat Completions接口，支持所有Qwen模型 |
| `POST /api/v1/services/aigc/text-generation/generation`（DashScope原生） | 百炼原生文本生成接口 |

**限流说明**：
- 按模型不同，QPS限制从 10~500 次/秒不等（取决于账户等级和模型）
- 支持 TPM（Tokens Per Minute）和 RPM（Requests Per Minute）配额控制
- 429 错误表示限流，需实现退避重试机制

| 评估结果 | 数量 | 占比 |
|----------|------|------|
| 完全支持（百炼原生） | 12 | ~14% |
| 部分支持（百炼提供基础能力，需自研增强） | 15 | ~17% |
| 不支持（完全自研） | 59 | ~69% |

**关键结论**：百炼平台主要提供**AI底层能力**（大模型推理、知识库RAG检索、智能体调用），但**不包含**在线客服系统、会员管理、渠道接入、报表、VIP专属路由、多模态推荐等上层业务功能。V2方案新增的VIP专属客服工作台和多模态推荐引擎均为完全自研。

### 4.3 数据流向

#### 常规用户AI对话流程（V2）

```
用户消息 → 渠道适配器 → API网关 → 会员等级识别(常规用户)
  │
  ├── 1. 调用百炼智能体应用API（识别意图）
  ├── 2. 调用百炼 Retrieve API（检索知识库）
  ├── 3. 多模态推荐引擎并行执行：
  │     ├── 从商户DB匹配相关商户
  │     ├── 从商品DB匹配相关商品
  │     ├── 从优惠券DB匹配可用优惠券
  │     ├── 从活动DB匹配当前活动
  │     └── 从导航DB获取位置导航
  ├── 4. 组装多模态响应（AI文本 + 推荐卡片）
  ├── 5. 流式返回AI回答 + 推荐卡片
  └── 6. 记录会话日志到MySQL
       │
       └── 用户要求人工：
             ├── 展示服务台电话号码
             └── 展示服务台地址导航引导
```

#### VIP用户专属客服流程（V2）

```
VIP用户消息 → 渠道适配器 → API网关 → 会员等级识别(VIP)
  │
  ├── 首次提问：
  │     ├── 1. 查询VIP专属客服绑定关系
  │     ├── 2. 查询专属客服状态（空闲/忙碌/离线）
  │     │
  │     ├── 空闲：
  │     │     ├── 直接分配给专属客服
  │     │     ├── 推送通知给专属客服（小程序工作台）
  │     │     └── 建立VIP-专属客服对话通道
  │     │
  │     └── 忙碌：
  │           ├── 计算排队位置 + 预计等待时间
  │           ├── 展示两个选项：
  │           │     ├── 选项A：排队等待
  │           │     └── 选项B：拨打专属客服电话
  │           └── 根据用户选择执行
  │
  └── 连接后（专属客服已接管）：
        ├── 用户后续提问 → 直接推送给专属客服
        ├── AI对话服务旁路（不经过AI）
        ├── 专属客服通过小程序工作台回复
        │     ├── 文本回复
        │     └── 推送多模态内容（商户/商品/活动/优惠券/导航数据卡片）
        └── 专属客服从后端数据库选择推荐数据推送
```

#### 知识库管理流程

```
管理后台上传文档 → 自研后端
  │
  ├── 1. 调用 ApplyFileUploadLease（获取上传租约）
  ├── 2. 上传文件到百炼临时存储
  ├── 3. 调用 AddFile（导入文件到数据连接）
  ├── 4. 轮询 DescribeFile（等待解析完成）
  ├── 5. 解析完成后，提交到知识库（SubmitIndexAddDocumentsJob）
  └── 6. 进入双人审核流程（自研）
        │
        ├── 审核人A通过 → 审核人B通过 → 发布上线
        └── 任一驳回 → 退回修改
```

## 7. 知识库管理复刻方案

本节详细说明如何在自研管理后台中，通过百炼API完整复刻知识库管理功能。（与V1保持不变）

### 7.1 文件上传完整流程

```
步骤1: 用户在管理后台选择文件上传
       │
       ▼
步骤2: 自研后端校验
       - 文件格式：PDF/DOC/DOCX/EXCEL/TXT/MD
       - 文件大小：≤50MB
       - 文件类型与解析器匹配性
       │
       ▼
步骤3: 调用 GetAvailableParserTypes
       - 输入：文件扩展名
       - 输出：可用解析器列表（如 PDF → [DASHSCOPE_DOCMIND, GENERAL]）
       │
       ▼
步骤4: 调用 ApplyFileUploadLease
       - 输入：category_id, workspace_id, file_name, md5, size
       - 输出：upload_url, upload_headers, lease_id
       │
       ▼
步骤5: 上传文件到百炼临时存储
       - HTTP PUT upload_url
       - 携带 upload_headers（X-bailian-extra, Content-Type）
       │
       ▼
步骤6: 调用 AddFile
       - 输入：workspace_id, lease_id, parser, category_id
       - 输出：file_id
       │
       ▼
步骤7: 轮询 DescribeFile
       - 查询文件解析状态
       - 状态流转：INIT → PARSING → PARSE_SUCCESS / PARSE_FAILED
       - 建议轮询间隔：5秒，超时60秒
       │
       ▼
步骤8: 解析成功后，记录到自研数据库
       - knowledge_base 表：记录 file_id, 分类, 标签, 状态(待审核)
       - 等待双人审核流程
```

**关键实现细节**：
- MD5计算：上传前计算文件MD5，用于 ApplyFileUploadLease 参数和去重检测
- 上传方式：使用预签名URL直接上传至百炼临时存储，不经过自研服务器中转（节省带宽）
- OSS替代方案：对于大文件批量导入，可使用 AddFilesFromAuthorizedOss 直接从OSS导入

### 7.8 双人审核流程（自研核心）

```
知识条目状态机：

[草稿] ──提交审核──→ [待审核A] ──A通过──→ [待审核B] ──B通过──→ [已发布]
                      │                   │                      │
                      └──A驳回──→ [已驳回]  └──B驳回──→ [已驳回]  └──下架──→ [已下线]

[已发布] ──编辑修改──→ [待审核A]  (重新走审核流程)
[已发布] ──到期自动──→ [已过期] → [已下线]

实现方式：
- 自研数据库 knowledge_base 表维护审核状态（review_status_a, review_status_b）
- 文件实际存储于百炼平台（仅当审核通过后才 SubmitIndexAddDocumentsJob 加入检索索引）
- 未通过审核的知识不参与AI检索

审核人A操作：
  PUT /api/v1/knowledge/{id}/review
  { "action": "approve/reject", "reviewer": "a", "comment": "..." }
  → 更新 review_status_a
  → 若approve且review_status_b已通过 → 调用 SubmitIndexAddDocumentsJob

审核人B操作：
  同上，更新 review_status_b
```

## 8. 百炼API限制与风险

### 8.1 API限流（Rate Limiting）

| API类别 | 限流阈值 | 影响 | 应对策略 |
|---------|----------|------|----------|
| 知识库检索（Retrieve） | **5次/秒** | 高并发场景下检索请求会被限流 | ①自研缓存层（Redis）缓存高频问题检索结果 ②实现指数退避重试 ③申请提升配额 |
| 数据连接管理API | 10次/秒 | 批量文件操作可能触发限流 | ①批量操作时控制并发 ②实现请求队列 |
| 知识库管理API | 10次/秒 | 管理后台并发操作可能触发限流 | 管理后台用户量少，一般不受影响 |
| Prompt工程API | 10次/秒 | 模板管理操作 | 管理后台低频操作，无影响 |
| 大模型推理（Chat Completions） | 按模型不同 10~500次/秒 | 高并发AI对话可能触发限流 | ①流式输出降低单次占用 ②申请提升配额 ③多模型负载分担 |
| 多模态交互网关 | 2026年4月28日起调整为 **10 QPS**（默认） | 图片/多模态请求受限 | ①按需申请提升配额 ②图片预处理降低请求频率 |

### 8.2 缺失的API能力

| 缺失能力 | 影响 | 替代方案 |
|----------|------|----------|
| **智能体创建/编辑API** | 无法通过代码自动化创建、修改智能体配置 | 通过百炼控制台手动配置，代码仅调用运行 |
| **工作流创建/编辑API** | 无法通过代码自动化编排工作流 | 通过百炼控制台手动编排工作流 |
| **FAQ问答对API** | 百炼知识库不支持FAQ式"问题→答案"直接映射 | 自研"补丁问答"表，在检索前优先匹配 |
| **相似问法管理API** | 百炼不支持配置相似问法 | 自研相似问法映射表 |
| **审核流程API** | 百炼无审核机制 | 自研双人审核状态机 |
| **权限管理API** | 百炼无应用级RBAC权限 | 自研权限体系 |
| **操作日志API** | 百炼不提供操作日志查询 | 自研日志系统 |
| **有效期管理API** | 百炼知识无有效期字段 | 自研有效期管理 |

### 8.3 检索延迟

| 指标 | 数值 | 说明 |
|------|------|------|
| Retrieve API响应时间 | 通常 200~800ms | 取决于知识库规模和top_k设置 |
| 大模型推理响应时间 | 首字 ~500ms~1.5s，完整生成 2~5s | 取决于模型选择、输入长度、输出长度 |
| 端到端AI响应时间 | 预计 1.5~4s | Retrieve + Prompt组装 + 模型推理的总和 |
| **3秒首次响应要求** | ⚠️ 边界 | 需优化：并行检索+流式输出+缓存 |
| **多模态推荐聚合时间** | 预计 +200~500ms | V2新增：从业务数据库聚合推荐数据的额外耗时 |

### 8.4 其他限制

| 限制项 | 数值 | 影响 |
|--------|------|------|
| 业务空间类目上限 | 500个/业务空间 | 一般够用，但需注意分类设计 |
| 文件大小限制 | 百炼临时存储单文件 ≤ 50MB | 与招标要求一致 |
| 文件删除 | 不可恢复 | 需自研逻辑删除/版本备份 |
| 切片修改 | 修改后需等待重新索引 | 有短暂延迟 |
| 知识库创建 | 异步任务，需轮询状态 | 不能立即使用 |

### 8.5 风险评估矩阵

| 风险项 | 可能性 | 影响 | 等级 | 缓解措施 |
|--------|--------|------|------|----------|
| Retrieve API限流（5次/秒） | 高 | 高 | **P0** | 自研Redis缓存高频问题结果，减少API调用 |
| 端到端响应超过3秒 | 中 | 高 | **P0** | 流式输出+并行检索+高频问题预计算 |
| 百炼平台可用性 | 低 | 高 | **P1** | 依赖SLA（99.9%），准备降级方案（缓存兜底回答） |
| 300万会话量Token成本 | 高 | 中 | **P1** | 优化Prompt长度，高频问题走缓存减少模型调用 |
| 智能体配置无法代码化管理 | 中 | 中 | **P2** | 使用Prompt模板API管理Prompt，其他配置文档化 |
| 知识删除不可恢复 | 中 | 中 | **P2** | 自研逻辑删除+定期备份 |
| 多模态网关限流（10 QPS） | 低 | 低 | **P3** | 按需申请提升配额 |
| **VIP专属客服单点故障（V2新增）** | 中 | 高 | **P1** | 专属客服离线时自动降级为排队+电话模式；支持备用专属客服绑定 |
| **多模态推荐延迟影响体验（V2新增）** | 中 | 中 | **P2** | 推荐数据预加载+Redis缓存；推荐响应超时则降级为纯AI文本 |
| **VIP任务推送延迟（V2新增）** | 低 | 高 | **P1** | 使用RocketMQ可靠消息+小程序订阅消息双通道推送 |

@Service
public class BailianDataConnectionClient {

    // 完整文件上传流程
    public FileUploadResult uploadFile(MultipartFile file, String categoryId) {
        // 1. 计算MD5和文件大小
        String md5 = calculateMD5(file);
        long size = file.getSize();
        String fileName = file.getOriginalFilename();
        
        // 2. 申请上传租约
        ApplyFileUploadLeaseResponse lease = 
            client.applyFileUploadLease(categoryId, workspaceId, fileName, md5, size);
        
        // 3. 上传到临时存储
        uploadToTempStorage(lease.getUrl(), lease.getHeaders(), file);
        
        // 4. 添加到数据连接
        AddFileResponse addResp = client.addFile(workspaceId, lease.getLeaseId(), 
                                                  parser, categoryId);
        
        // 5. 轮询等待解析完成
        String fileId = addResp.getFileId();
        awaitParseComplete(fileId);
        
        return FileUploadResult.builder()
            .fileId(fileId)
            .fileName(fileName)
            .status("PARSE_SUCCESS")
            .build();
    }
    
    // 轮询解析状态（最多12次，间隔5秒）
    private void awaitParseComplete(String fileId) {
        for (int i = 0; i < 12; i++) {
            FileStatus status = client.describeFile(workspaceId, fileId).getStatus();
            if ("PARSE_SUCCESS".equals(status)) return;
            if ("PARSE_FAILED".equals(status)) throw new ParseException();
            Thread.sleep(5000);
        }
        throw new TimeoutException("文件解析超时");
    }
}

### 10.3 检索优化策略

为满足"首次响应 ≤ 3秒"的要求，实施以下优化：

#### 10.3.1 三级缓存架构

```
用户提问
    │
    ▼
┌─────────────────────────────────┐
│ L1: 精确匹配缓存（Redis Hash）    │  → 命中：直接返回（< 50ms）
│ 精确问题 → 标准答案              │
└─────────────────────────────────┘
    │ 未命中
    ▼
┌─────────────────────────────────┐
│ L2: 语义检索缓存（Redis）         │  → 命中：使用缓存的检索结果（< 200ms）
│ query的MD5 → 检索结果 + 分数     │  TTL: 30分钟
└─────────────────────────────────┘
    │ 未命中
    ▼
┌─────────────────────────────────┐
│ L3: 实时检索（百炼Retrieve API）  │  → 正常调用（200~800ms）
│                                 │
└─────────────────────────────────┘
```

#### 10.3.2 并行化优化（V2增强）

```
传统串行：
  检索知识库(500ms) → 组装Prompt(10ms) → 调用模型(1500ms) = 2010ms

并行优化：
  [检索知识库(500ms)] 同时
  [加载对话历史(50ms)] → Prompt组装(10ms) → 调用模型(1500ms) = 1610ms

V2多模态并行优化：
  [检索知识库(500ms)] 同时
  [多模态推荐引擎查询(300ms)] 同时
  [加载对话历史(50ms)] → Prompt组装(10ms) → 调用模型(1500ms) = 1610ms
  
  首字响应时间：检索 + Prompt组装 + 模型首字 ≈ 500 + 10 + 300 = 810ms ✅ < 3s
  推荐数据异步加载，不阻塞首字响应
```

## 15. 结论与建议

### 15.1 方案可行性结论

**基于百炼平台的V2方案完全可行**，但需要明确以下定位：

1. **百炼平台定位**：提供AI底层能力（大模型推理、知识库RAG、智能体编排），是"引擎"而非"完整产品"
2. **自研范围**：约72%的业务功能（在线客服、会员、渠道、报表、VIP专属服务、多模态推荐等）需完全自研
3. **知识库管理**：可通过百炼API完整复刻管理功能，但需额外开发审核、版本、有效期等业务逻辑
4. **性能要求**：首次响应 ≤ 3秒需通过缓存优化、流式输出、并行化等手段保障
5. **V2核心差异**：常规用户走AI多模态推荐 + 电话/导航引导；VIP用户走专属客服直连/排队，所有后续对话由专属客服通过小程序工作台处理

### 15.2 V2方案核心变更总结

| 变更点 | 描述 | 影响 |
|--------|------|------|
| 常规用户转人工 | ❌ 不再支持在线转接，仅提供服务台电话+地址导航 | 简化常规用户服务路径，降低在线坐席成本 |
| VIP用户专属路由 | ✅ 首次提问即触发专属客服状态检查，空闲直连/忙碌排队+电话选项 | 提升VIP用户体验，确保专属服务 |
| 专属客服工作台 | ✅ 新增小程序工作台，支持多模态内容推送 | 为专属客服提供完整服务能力 |
| 多模态推荐引擎 | ✅ 常规用户AI响应聚合商户/商品/优惠券/活动/导航推荐卡片 | 丰富常规用户AI响应维度 |
| VIP任务推送 | ✅ VIP提问实时推送至专属客服小程序 | 确保VIP需求及时响应 |

### 15.3 核心建议

#### ✅ 建议采用百炼平台的理由

1. **开发周期缩短**：AI底层能力（模型、RAG、向量化）无需自研，节省约40%的开发时间
2. **技术风险降低**：百炼平台提供成熟的模型服务和知识库能力，避免自研RAG系统的技术风险
3. **成本可控**：按量计费，无需维护向量数据库和模型推理服务器
4. **持续升级**：百炼平台持续更新模型和RAG能力，自动享受技术红利
5. **V2方案优势**：VIP专属服务与AI能力分离，VIP服务不依赖AI稳定性

#### ⚠️ 需要注意的事项

1. **智能体/工作流配置需固化**：建议在项目早期就在百炼控制台完成智能体和工作流配置，形成配置文档，避免后期调整影响进度
2. **Prompt模板化管理**：使用百炼Prompt Template API管理Prompt，便于版本控制和A/B测试
3. **API限流提前应对**：Retrieve API仅5次/秒，必须实现缓存层。建议在项目启动后立即联系阿里云提升配额
4. **知识库管理后台是重点**：这是与百炼平台对接的核心，建议优先开发
5. **V2重点：专属客服工作台**：这是VIP用户体验的核心，需重点投入资源开发小程序工作台端
6. **V2重点：多模态推荐引擎**：需对接杭州大厦现有业务数据库（商户/商品/优惠券/活动/导航），数据对接需尽早启动
7. **V2重点：VIP路由稳定性**：专属客服单点故障需有降级方案（备用客服、自动转电话）
8. **等保测评提前启动**：等保二级测评周期较长（1-2个月），建议在方案阶段就开始准备

#### ❌ 不建议的做法

1. **不要在代码中动态创建/修改智能体**：百炼无此API，强行通过控制台操作接口（非公开）维护风险极高
2. **不要依赖百炼做会话管理**：百炼智能体的会话记忆仅限于单次对话上下文，不支持跨会话记忆、转人工、排队等业务逻辑
3. **不要忽略高频问题缓存**：300万会话量下，不做缓存会导致大量重复API调用，成本飙升且延迟增加
4. **不要将常规用户转人工做复杂**：V2已明确常规用户仅电话+导航，不要额外实现在线转接功能
5. **不要让VIP排队体验变差**：必须提供排队位置展示和预计等待时间，且随时可切换到电话模式

### 15.4 技术选型替代方案评估

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **百炼平台 + 自研**（当前方案） | 开发快、成本低、技术成熟 | VIP服务、推荐引擎等需自研 | ⭐⭐⭐⭐⭐ |
| 完全自研（LangChain + Milvus + 开源模型） | 完全自主可控 | 开发周期长、技术风险高、维护成本高 | ⭐⭐ |
| 商业客服SaaS（瓴羊/智齿/沃丰） | 功能完整、上线快 | 定制能力受限、年费高、数据不在自己控制下、V2定制需求难以满足 | ⭐⭐ |
| 百炼平台 + 第三方客服系统对接 | 减少自研量 | 第三方系统适配成本、集成复杂度、V2 VIP专属流程难定制 | ⭐⭐⭐ |
