### 行业调研关键发现

- **三层分级服务模式**已成为行业标准：AI机器人(70-85%分流) → AI助理辅助人工 → 人工专家坐席
- **大模型选型**：DeepSeek性价比最优（约GPT-4的1/10成本），通义千问与阿里云生态深度集成
- **RAG架构**是客服场景标配，优于纯LLM微调（知识更新灵活、可溯源、幻觉风险低）
- **高并发架构**：WebSocket网关集群 + 消息队列削峰 + K8s弹性扩缩容
- **行业案例**：王府井(网易云商+DeepSeek，85%AI分流)、天虹(灵智+天润融通，70%AI分流)
- **主流厂商**：瓴羊Quick Service、智齿科技、沃丰科技、网易七鱼等

#### 第3章 产品实现方案（补充内容）
- **3.1 三层分级服务模型**：AI机器人 → AI助理辅助人工 → 人工专家坐席
- **3.2 管理后台页面设计**：工作台Dashboard、知识库管理(分类/条目/编辑器/模拟测试/审核)、在线客服工作台(会话列表/对话区/信息辅助区)、机器人配置、会员画像、数据看板、权限管理
- **3.3 C端交互设计**：首次进入(欢迎语+快捷入口)、AI对话(富媒体卡片)、转人工(过渡页+排队提示)、人工服务(多模态)、留言场景、满意度评价
- **3.4 核心业务流程**：用户进线分流流程、知识库管理流程、会话转接流程

#### 第4章 技术架构方案（补充内容）
- **4.1 整体分层架构**：接入层 → 网关层 → 服务层(微服务) → AI引擎层 → 数据层 → 基础设施层
- **4.2 技术选型**：
  - 大模型：DeepSeek-V3(主) + 通义千问(备)，通过阿里云百炼平台统一管理
  - RAG：百炼RAG + 自研增强，向量数据库Milvus
  - 后端：Spring Boot 3 + Spring Cloud Alibaba
  - 前端：Vue 3 + Element Plus(后台) / Vant 4(C端)
  - 消息队列：RocketMQ
  - 数据库：RDS MySQL + Redis + Milvus + OSS
  - 容器：阿里云ACK(K8s)
- **4.3 RAG四层流水线**：数据接入 → 文档处理与索引 → 检索与重排 → 生成与输出
- **4.4 高并发架构**：WebSocket网关集群(单Pod 5000连接) + RocketMQ削峰 + K8s HPA弹性扩缩
- **4.5 云端部署方案**：阿里云全套（百炼+ACK+RDS+Redis+Milvus+OSS+SLB+SLS）
- **4.6 安全合规设计**：VPC隔离、TLS加密、AES-256存储加密、WAF、等保二级、日志留存≥180天

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 大模型选型 | DeepSeek-V3(主) + 通义千问(备) | DeepSeek性价比最优(约GPT-4的1/10)，通义千问与阿里云生态无缝集成 |
| AI架构 | RAG（非纯LLM微调） | 知识更新灵活、可溯源、幻觉风险低，适合客服高准确性要求 |
| 部署模式 | SaaS + 定制化混合 | 核心客服用成熟SaaS平台，AI能力和业务特色功能定制开发 |
| 云平台 | 阿里云 | 百炼平台原生RAG能力、Milvus向量检索、ACK容器编排、微信生态集成 |
| 前端技术 | Vue 3 | 成熟生态，Element Plus/Vant组件丰富，适合中后台+移动端 |
